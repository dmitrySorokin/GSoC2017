New methods for trajectory integration in Geant4

Introduction

In accelerator simulation one needs to construct tracks of charged particles. Presents of an electromagnetic field curves tracks due to the Lorentz force. Mathematically speaking the track is described by an initial value problem (IVP):

(1)

If the field is not uniform the tracks should be computed numerically by solving the above equation. In simulation of real experiments the field is evaluated by table lookup and interpolation thus function evaluations are expensive. So for simulation of complex High Energy Physics experiments we need robust and time efficient track integration method.

Challenges:

  • very robust integration of different equations (with a user provided field) for different length scales, without failures.  
  • high cost of field evaluation, but also change of momentum (currently only at end of a ‘physical’ step) due to interactions.  This imposes some limit on the ability to reuse previous derivatives (multi-step methods.)
  • need to find the intersection point between a curved trajectory and the surfaces of user’s volumes - which we seek to address using RK methods with interpolation to provide arbitrary intermediate points.
  • Observed that the step size sometimes oscillates, leading to the idea to try different step-size control methods.

This year’s GSoC project is dedicated to new step-size control algorithms, steppers for oscillatory problems and the revision of drivers for RK to enable use of FSAL (and pave way for interpolation).

Part 1: New methods for step size control

Context

For nonuniform field right hand side (RHS) of equation (1) can change significantly in one region and slightly in an other. To integrate accurately in the regions with strong RHS changes one need to set a very small integration steps which will slow down integration in the regions with small RHS changes. To solve this problem one need to use adaptive step size control. Step size control algorithms can adjust the step size dynamically to satisfy the error tolerance (TOL) so it would be small enough in regions with strong field changes and large in regions with small field changes.

Part 3 - Using FSAL in integration

The revised design

The library contains the following main classes:

  1. Stepper (G4MagIntegratorStepper - base class)

2.   Driver (G4VIntegrationDriver - base class)

3.  ChordFinder (G4ChordFinder)

The Stepper class increments array of independent variables, calculates errors and estimates maximum distances between the chord and the curves. The Driver class uses the stepper object to integrate the equation of motion with the adaptive step size control. The ChordFinder class uses the Driver object to integrate the equation of motion over steps satisfying chord criterion. This classes and their associations are represented in the uml diagram below.

Fig. 2. Uml diagram of the magneticfield package..

Addition of the new abstract driver base class enables usage of different driver algorithms: Bulirsch-Stoer, Standard (G4MagInt_Driver), FSAL.

Virtual function overhead may affect overall performance of the integration process. To decrease this effect the driver classes are implemented as templates. This eliminates overhead of the the virtual function call to the stepper methods and allow usage of steppers which don’t inherit from the base class.

FSAL steppers

Context

  • A Runge-Kutta method has a First Same As Last (FSAL) property If the last stage is evaluated at the same point as the first stage of the next step. 
  • Using this property one can get one RHS evaluation ‘for free’.  

The work done

Current design of a FSAL stepper:

The RK547FEq1 stepper can be applied as FSAL and non-FSAL stepper. Because we created the template FSAL driver class we don’t need to add an additional base driver class for the FSAL steppers.

Work [2] presents three FSAL Runge-Kutta steppers (RK547FEq1,2,3) with stable equilibrium properties. These steppers were implemented as described above and tested.

RK547FEq1 is more efficient than the classical RK4 method in number of field evaluations (Fig.6). Field evaluations decrease is 16%, 36%, 44% for run2a.mac, run2b.mac and run2c.mac accordingly.

This allows to save one field evaluation at each step. In case of seven stage method (RK547FEq1) this property can give up to 1/7  14% decrease in number of field evaluations. It is upper bound estimation because it doesn’t take into account processes like scattering, energy loss which change momentum of the particle and thus reset the integration process.

G4FSALIntegrationDriver is using FSAL property only in the AccurateAdvance method. We could use this property in the QuickAdvance method too but it will require certain changes in G4ChordFidner class.
